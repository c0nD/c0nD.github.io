CRK Guild Battle BOT: Usage Documentation

Check out our community webpage!

Introduction

    Welcome to the documentation page of the Fluffy! This is an open source Discord bot that allows users to keep track of their hits for Cookie Run Kingdom. The bot is designed to streamline the process of tracking hits and provides accurate and real-time data to the leaders.
    With Fluffy, users can call out their hits using commands, and the bot will automatically track all their data, including the split status, total damage dealt, and more. The leaders can access this data in a CSV file, which can be used for analysis and decision making. Whether you are a a guild member or guild leader, Fluffy provides an excellent tool for managing hits, tracking progress, and achieving your goals. In this documentation, we will cover everything you need to know to get started with Fluffy including installation, setup, and usage.

Installation

The install is not as straight forward as you would think. Because different Discord servers may be set up differently, you will need to set up the bot in a manner that will mesh with your server's setup. Firstly, head over to the developer discord applications portal here. Use this website to create your bot, and set up it's permissions/trust settings, and create a link to add it to your server. Once you have a bot in your server, go ahead and generate a token for the bot. DO NOT SHARE THIS TOKEN WITH ANYBODY. The token is essentially your bot's password to be able to run. If someone gets a hold of that token, they can run whatever code they want from your bot in any server it resides in.

Next, set up any specified channels for callouts. By default, the bot's code is setup to run on the channel names "avatar", "living_abyss" and "dragon" for the respective bosses for the in-game bosses. After you've set these channels up, you can now follow the README to complete your setup. You will need to change the IDs of certain variables in final_vars.txt, create a guild.txt file that will contain the ID of your discord server, and finally put the token from earlier into a file called tkn.txt (all in the same directory).

After this, to keep your bot running, it is reccomended to use pm2 on a server to keep the boss running. Once you have your bot running, all the commands presented below with their use cases can be used to their fullest potential! Good luck, and happy hitting.

Commands

User Commands

  • /hit [damage] - Hits the current boss in the text channel for damage amount. This does consume a ticket on usage. It is recommended to separate damage values with _ (123_345_789).
  • /resume_hit [damage] - Hits the current boss in the text channel for damage amount. This does not consume a ticket on usage. It is recommended to separate damage values with _ (123_345_789).
  • /killed - Kills the current boss in the text channel. This does consume a ticket on usage.
  • /bonus_kill - Kills the current boss in the text channel. This does not consume a ticket on usage.
  • /hp - Sends the current boss's hp.
  • /boss_help - Sends link of documentation page to user

    • Admin Commands

  • /admin_hit [damage] - Hits the boss for damage HP. (Does not count as a real hit)
  • /admin_kill - Kills the boss and increments level. (Does not count as a real hit)
  • /admin_revive - Revives the boss and decrements the level. (Does not count as a real hit)
  • /insert_hit [user_id] [damage] [split] - Inserts a hit for another user
  • /create_boss [guild] - Creates a boss tied to the current text channel with a guild attribute from the guild family. Initializes boss to max hp of a level 1 boss.
  • /delete_boss - Deletes the current boss in the channel. Removes all hits linked to the current boss as well.
  • /insert_boss [guild] [level] [hp] - Usage same as /create_boss, but guild, level, and hp can all be specified upon creation.
  • /send_csv [guild] - Sends a sorted .csv file of the specified guild.
  • /send_backup_csv [guild] - Sends a sorted .csv file for the specified guild after backup json file is loaded.
  • /load_json - Will load the current JSON backup file into the bosses dictionary. Note: this will not make the previously saved bosses hittable. Check usage for more information.
  • /insert_hit [user_id] [damage] [ticket_used] - Inserts a hit for a specific user.
  • /insert_kill [user_id] [ticket_used] [split] - Inserts a kill for a specific user.

Usage

User Usage

  • You will only need to understand how to use 4 commands:
    • /hit, /resume_hit, /killed and /bonus_kill

Note: the command /bonus_kill can be thought of as solo by some, and /killed can also be thought of as sweep by some.

It is recommended to separate damage numbers with underscores to ensure you type the correct number. Ex: /hit 123_456_789 is equivelant to /hit 123456789.

The main differences you'll need to understand as a user interacting with the bot is one key feature: "Does this command use a ticket or not?" The commands /hit and killed will both use a ticket when called. The commands /resume_hit and /bonus_kill will not use a ticket when called.
Because of this difference, you want to use /hit and /killed on attacks when you do not resume the hit. If you do resume the hit, use the respective commands: bonus_kill and resume_hit.

Usage Examples:

  • You hit the boss and do 23,500,000 damage: /hit 23_500_000
  • You kill the boss and do 3,250,000 damage on the resumed hit: /killed then /resume_hit 3_250_000
  • You killed the boss twice and did 500,000 damage on the final resumed hit: /killed then /bonus_kill then /resume_hit 500_000 (note bonus_kill's main usage is clearing multiple boss levels with a single ticket)

Note: most cases, you will either have this sequence: /killed -> /resume_hit [damage] or simply /hit [damage]. The use case for using /bonus_kill will most likely be limited to players who solo a boss.

Admin Usage

Boss Creation/Deletion in a Channel

In the channels specified for the bosses, you need to initialize the boss. To do this, you can either use /create_boss or /insert_boss [guild] [level] [hp]. The use case for /create_boss will initialize the boss to level 1 at max hp. (It is reccomended to do this one during tally day to initialize the bosses when they're available to be hit.) The use case for using /insert_boss [guild] [level] [hp] is if the bot went down, and you need to insert a boss at a specific state. DO NOT USE IT TO CHANGE THE HP/LEVEL -- PLEASE USE ADMIN PREFIXED COMMANDS TO CHANGE THE HP OR LEVEL. USING IT TO CHANGE STATS WILL RESULT IN ALL HIT DATA BEING LOST FOR THAT BOSS.

Using /delete_boss will delete the boss from the channel along with all data attached to it (ie. hits). When a season is done, you will want to delete all bosses from their respective channels and recreate them.

Admin Prefixed Commands + Inserting a hit

Admin prefixed commands (/admin_hit, /admin_kill) work exactly the same as their non-admin prefixed counterparts (note that admin_hit can also accept negative numbers to heal the boss)(/hit and /killed) with the exception that it will not log a hit in the hit list attached to the boss. /admin_revive will restore the boss at the previous level. If someone accidentally killed the boss, you can revive the boss and hit it down to the correct hp with these admin prefixed commands. If someone needs a hit or kill inserted for them that you would like to be recorded, you can insert one for them using /insert_hit [user_id] [damage] [split] or /insert_kill [user_id] [ticket_used] [split]. Grab their discord user id (using developer mode in discord -- right click their profile in the side bar and click 'Copy ID'), and insert the damage/kill you'd like for them to have, as well as a true/false value for if the hit was split or not and if a ticket was used.

Data Aggregation

/send_csv [guild] will send a CSV file containing sorted data from the specified CRK guild bosses. This will most likely be used at the end of each season. This data can then be sorted in Google Sheets or Microsoft Excel.

(EMERGENCY COMMANDS)
The command /load_json should only be used in a very specific scenario. This command will take the last loaded backup of the database and load it into the boss_dictionary so that you can call /send_backup_csv [guild] to get the data before the bot went down. After you use /load_json you should use /send_backup_csv to get the data from before the crash. (Note that this will not set up the bosses in the channel -- you will need to use /insert_boss [guild] [level] [hp] in order to set up the current boss(es).)
